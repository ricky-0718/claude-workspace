# ハンドオフ: スキル/コマンドをエージェントに割り当てる

## 目的
現在 `.claude/commands/` にあるコマンド（スキル）を、対応するエージェント（`.claude/agents/`）に持たせる。
エージェントが召喚された時に、自分の持つスキルの手順・ルールを自動的に把握して動けるようにする。

## 背景
- 現状、コマンドとエージェントが別々の動線になっている
- 例: `/utage-check` は司令塔が直接実行するが、本来は customer-manager の専門領域
- エージェントにスキルを持たせることで、召喚時に自動でスキルのワークフローに従って動ける

## 実装方針の検討事項

### 方法1: エージェント定義にスキル内容を埋め込む
- `.claude/agents/customer-manager.md` の中に utage-check のワークフローを記載
- メリット: エージェント召喚時に自動で読み込まれる
- デメリット: エージェント定義が肥大化する

### 方法2: エージェント定義からスキルファイルを参照
- エージェント定義に「このスキルを持つ: `/utage-check`」と記載し、実行時にコマンドファイルを読む
- メリット: スキルとエージェントの分離が保たれる
- デメリット: 参照の仕組みが必要

### 方法3: コマンド側にエージェント召喚を組み込む
- `/utage-check` 実行時に customer-manager エージェントを召喚する形にする
- メリット: 既存構造を大きく変えない
- デメリット: コマンド→エージェント→スキルの間接呼び出しになる

→ **セッション開始時にユーザーと方針を決めてから実装すること**
