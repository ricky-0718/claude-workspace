# スプレッドシート設定リファレンス

## 命名規則

スプレッドシート名: `【{キャンペーン名}】広告分析ダッシュボード`

例: `【LP-Bテスト】広告分析ダッシュボード`

## 作成するシート

### 1. Meta広告_生データ

GASがAPIデータを書き込むメインシート。

**ヘッダー (A1:G1):**

| 列 | ヘッダー名 | データ型 | 説明 |
|----|-----------|---------|------|
| A | 日付 | YYYY-MM-DD | APIの date_start |
| B | 広告費 | 数値（円） | spend |
| C | インプレッション | 整数 | impressions |
| D | リーチ数 | 整数 | reach |
| E | クリック数 | 整数 | inline_link_clicks |
| F | LINE登録数 | 整数 | offsite_conversion.fb_pixel_complete_registration |
| G | フリークエンシー | 小数 | frequency |

**書式設定:**
- A列（日付）: 列幅 120px、書式 `yyyy-mm-dd`
- B列（広告費）: 列幅 100px、書式 `#,##0`（通貨）
- C-F列（整数）: 列幅 120px、書式 `#,##0`
- G列（フリークエンシー）: 列幅 120px、書式 `#,##0.00`
- ヘッダー行: 太字、背景色 `#4285f4`（Googleブルー）、文字色 白、固定

### 2. 日次集計

日次の集計ビュー。基本構造のみ作成（集計数式は後で追加可能）。

**ヘッダー (A1:H1):**

| 列 | ヘッダー名 | 説明 |
|----|-----------|------|
| A | 日付 | 集計対象日 |
| B | 広告費 | 当日の広告費 |
| C | CPA | 広告費 ÷ LINE登録数 |
| D | CTR | クリック数 ÷ インプレッション × 100 |
| E | CVR | LINE登録数 ÷ クリック数 × 100 |
| F | LINE登録数 | 当日の登録数 |
| G | クリック数 | 当日のクリック数 |
| H | 累計広告費 | 開始日からの累計 |

**書式設定:**
- ヘッダー行: 太字、背景色 `#34a853`（Googleグリーン）、文字色 白、固定

### 3. KPIダッシュボード

KPI可視化用。基本ラベルのみ作成。

**初期ラベル:**
- A1: `KPIダッシュボード`（太字、フォントサイズ 16）
- A3: `期間:`
- A4: `総広告費:`
- A5: `総LINE登録数:`
- A6: `平均CPA:`
- A7: `平均CTR:`
- A8: `平均CVR:`

**書式設定:**
- A1: フォントサイズ 16、太字
- A列: 列幅 150px

## ブラウザ操作でのシート作成手順

1. `sheets.new` で新規スプレッドシートを作成
2. タイトルをクリックして命名規則に従い名前を設定
3. デフォルトの「シート1」を右クリック → 「名前を変更」 → `Meta広告_生データ`
4. A1:G1 にヘッダーを入力
5. 「+」ボタンで新しいシートを追加 → `日次集計` に命名
6. 「+」ボタンで新しいシートを追加 → `KPIダッシュボード` に命名
7. 各シートのヘッダーを入力

## 注意事項

- シート名は正確に入力すること（GASのCONFIG.SHEET_NAMEと一致する必要がある）
- ヘッダー行は1行目に固定（GASは2行目からデータを書き込む）
- 日付列のフォーマットが `yyyy-mm-dd` であることを確認（GASの重複チェックで使用）
