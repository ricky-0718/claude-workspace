# /review-customer — 顧客視点レビュー

直前の作業内容、または指定された施策・成果物を**顧客（見込み客）の目線**でレビューしてください。

## 判断基準（必ず読み込んでからレビューすること）

**`.claude/skills/customer-review-criteria.md` を最初に読み込む**こと。このファイルには：
- 実際の面談12件から抽出した顧客プロファイル
- 5大不安（費用・保護者・信頼・能力・就職）の出現頻度と解決パターン
- LP離脱データ（80%がFVで離脱、平均滞在28秒）
- 成約トリガーと失注トリガーの実データ
- 意思決定プロセスの構造

このSkillsファイルの実データに基づいてレビューする。「想像」ではなく「実際に起きたこと」との照合。

## レビューの問いかけ（5つ）

1. **3秒テスト**: 80%がFVで離脱する現実を踏まえ、パッと見て「高校生/保護者の自分向け」と分かるか？
2. **5大不安チェック**: 費用（12件中10件で出現）・保護者（失注5件中5件の原因）・信頼・能力・就職の各不安に対処があるか？
3. **費用フレーム**: 149万が「日本の大学1,080万との差額500万お得」という比較で提示されているか？
4. **保護者導線**: 面談に参加していない保護者への説明導線が設計に含まれているか？（失注の最大原因）
5. **同類事例**: 「自分と同じ状況の人が成功した」実名事例が提示されているか？

## レビュー時の態度

- **素直に反応する**: 「ここで離脱する」「これは意味がわからない」「これは刺さった」を正直に言う
- **専門用語を指摘する**: 顧客が知らない用語があれば「これ何？」と指摘する
- **感情で反応する**: マーケティング的な正しさではなく「この人に頼みたいと思うか」で判断する
- **スマホ前提**: 長い文章は読まない。スクロールが多すぎたら離脱する

## 出力フォーマット

```
━━━ Customer Review ━━━

■ 対象: [レビュー対象]

■ 第一印象（3秒テスト）:
  [正直な第一印象。「自分向けだ」と思えたか？]

■ 信頼スコア: [1-10] / 10
  信頼を感じた要素: [具体的に]
  信頼が足りない要素: [具体的に]

■ 不安チェック:
  ・費用の不安: [解消された / まだ不安 / 触れられていない]
  ・難易度の不安: [解消された / まだ不安 / 触れられていない]
  ・就職の不安: [解消された / まだ不安 / 触れられていない]
  ・言語の不安: [解消された / まだ不安 / 触れられていない]

■ 行動を止めるポイント:
  [「ここで手が止まる」「ここで離脱する」を具体的に]

■ 競合との差別化:
  [「他ではなくここを選ぶ理由」が伝わったか]

■ 顧客としての本音:
  [LINE登録する / もう少し調べる / 閉じる のいずれかと、その理由]
```
