# YouTube 文字起こし取得ワークフロー

YouTube動画のトランスクリプト（字幕テキスト）をMCPサーバー経由で取得し、ファイル保存や要約を行う。

## ステップ1: 動画情報を収集

ユーザーに以下を確認（AskUserQuestion推奨）:

1. **YouTube URL**（必須）
   - 通常URL: `https://www.youtube.com/watch?v=...`
   - 短縮URL: `https://youtu.be/...`
   - Shorts: `https://www.youtube.com/shorts/...`
   - 動画IDのみでもOK

2. **言語**（デフォルト: `ja`）
   - 日本語: `ja`
   - 英語: `en`
   - その他の言語コード

3. **タイムスタンプ**（デフォルト: なし）
   - あり: `[0:05] テキスト` 形式で時間付き
   - なし: テキストのみ

4. **用途**（任意。後続処理の判断に使う）
   - 保存のみ
   - 要約してほしい
   - 特定のトピックを抽出してほしい

## ステップ2: 文字起こし取得

MCPサーバーのツールを呼び出す:

```
ツール: youtube-transcript サーバーの get_transcript
パラメータ:
  - url: ユーザーが指定したURL
  - lang: 指定された言語コード（デフォルト "ja"）
  - include_timestamps: タイムスタンプ要否（デフォルト false）
  - strip_ads: true（広告・スポンサー部分を除外）
```

### エラー時の対応
- **字幕が存在しない場合**: 言語を `en` に変えて再試行。それでもダメならユーザーに報告
- **URLが無効な場合**: 正しい形式を案内
- **ネットワークエラー**: 再試行を提案

## ステップ3: 結果の保存

取得したテキストをファイルに保存する。

**保存先:** `youtube-transcripts/` フォルダ（なければ作成）
**ファイル名:** `{YYYY-MM-DD}_{動画タイトルまたはID}.md`

### ファイルフォーマット:
```markdown
# {動画タイトル}

- **URL**: {YouTube URL}
- **取得日**: {今日の日付}
- **言語**: {言語コード}

---

{文字起こしテキスト}
```

## ステップ4: 後続処理（用途に応じて）

### 要約を求められた場合
文字起こし全文を読み、以下の構成で要約を生成:
- **概要**（3〜5行）
- **主要ポイント**（箇条書き）
- **キーワード**

### 特定トピック抽出を求められた場合
指定されたトピックに関連する部分を抜き出して整理する。

### 保存のみの場合
ファイルパスをユーザーに伝えて完了。

## 注意事項
- MCPサーバー `youtube-transcript` が接続されている必要がある
- トランスクリプトには25,000トークンの上限がある（MCP制限）
- 長時間動画の場合、テキストが切り詰められる可能性あり
