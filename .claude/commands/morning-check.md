# /morning-check — 朝の一括チェックコマンド

Gmail・Asana・UTAGEのLINEチャットを**並列Agent**で同時チェックし、優先度順にアクションリストを出力する。

## 注意事項（必ず遵守）
- **Edge専用**。Chrome禁止。`switch_browser` 禁止
- LINE返信ルール: 語尾に「ね」を使わない。丁寧だがフランクすぎない文体
- `memory/utage-line-feedback.md` を先に読み、過去のフィードバックを反映する
- `memory/writing-rules.md` を先に読み、文章生成ルールを適用する

## ワークフロー

### Phase 1: 並列データ収集（3つのAgentを同時起動）

以下の3つのAgentを**すべて同時に**起動する（1つのメッセージで3つのAgent tool callを送る）。

#### Agent A: Gmail チェック（subagent_type: general-purpose）
```
Gmailの未読メールを確認し、対応が必要なものを抽出してください。

手順:
1. `gmail_search_messages` で `is:unread newer_than:3d` を検索（maxResults: 30）
2. 以下のカテゴリに分類:
   - 🔴 要返信: 人からの直接メール（取引先、顧客、パートナー）
   - 🟡 要確認: 銀行通知、送金完了、請求書、セキュリティ通知
   - ⚪ スキップ: ニュースレター、プロモーション、自動通知（NewsPicks, TikTok等）
3. 🔴のメールは `gmail_read_message` で本文を読み、返信案を生成
4. 🟡のメールは要点を1行でまとめる

出力フォーマット:
---
## Gmail サマリー（YYYY/MM/DD）

### 🔴 要返信（X件）
1. **件名** — 送信者 / 受信日時
   - 要点: ...
   - 返信案: ...

### 🟡 要確認（X件）
1. **件名** — 送信者 / 要点1行

### ⚪ スキップ（X件）
- 件数のみ報告
---
```

#### Agent B: Asana タスクチェック（subagent_type: general-purpose）
```
Asanaの自分に割り当てられたタスクと、今週期限のタスクを確認してください。

手順:
1. `search_tasks_preview` で自分（assignee_any: "me"）の未完了タスク（completed: false）を検索
2. 期限が今週中（due_on_before で7日後を指定）のタスクを特にフラグ
3. `get_status_overview` で「日報」プロジェクトの最新ステータスも確認

出力フォーマット:
---
## Asana サマリー（YYYY/MM/DD）

### 🔥 今日期限（X件）
1. タスク名 — プロジェクト名 / 期限

### 📅 今週期限（X件）
1. タスク名 — プロジェクト名 / 期限

### 📋 その他の未完了タスク（X件）
1. タスク名 — プロジェクト名 / 期限（or 期限なし）
---
```

#### Agent C: UTAGE LINEチャットチェック（subagent_type: general-purpose）
```
UTAGEのLINEチャットを確認し、未読・要対応メッセージをリストアップしてください。

注意: Edge専用。Chrome禁止。switch_browser禁止。

手順:
1. `tabs_context_mcp` で現在のタブ情報を取得
2. 新規タブを作成し、UTAGEチャットページに移動: https://utage-system.com/account/hVTTIqwMtevI/line/talk
3. ページ読み込み後、`read_page` でチャットリストを確認
4. 赤い「要対応」タグ、未読バッジのある友だちを特定
5. 各チャットをクリックして最新メッセージ内容を確認
6. メモ欄も確認し、前回の返信案や文脈を把握
7. 2つ目のアカウント「休止」は無視する

出力フォーマット:
---
## UTAGE LINEチャット サマリー（YYYY/MM/DD）

### 要対応（X件）
1. **名前** — 最新メッセージ要約 / 前回メモの有無
   - 最新メッセージ: 「...」
   - メモ: ...（あれば）

### 未読（X件）
1. **名前** — 最新メッセージ要約

### 対応不要
- 件数のみ報告
---
```

### Phase 2: 結果集約＆優先度判定

3つのAgentの結果を集約し、**全アイテムを横断的に優先度ソート**する。

優先度基準（上から順）:
1. **P1 - 即対応**: 顧客からの直接質問、契約関連、期限が今日のタスク
2. **P2 - 今日中**: 要返信メール、UTAGE要対応、今週期限タスク
3. **P3 - 確認のみ**: 銀行通知、セキュリティ通知、情報共有
4. **P4 - スキップ**: ニュースレター、プロモーション

### Phase 3: 返信案＆アクションプラン生成

P1・P2の各アイテムについて:
- **メール**: 返信案を生成（Agent Aが既に生成済みならそのまま使用）
- **LINEチャット**: `memory/utage-line-feedback.md` のフィードバックを反映した返信案を生成
- **Asanaタスク**: 具体的な遂行手順（何をどの順番でやるか）を記載
