# /mendan — 面談後フォロー自動化コマンド

面談後の分析・フォロー業務を一括実行します。

## ステップ1: 面談者情報を収集

以下を質問してください（AskUserQuestion推奨）:
- 面談者の名前（フルネーム）
- 学年（高1/高2/高3/大学生/社会人/保護者）
- 同席者（本人のみ/母同席/父同席/両親同席）
- 面談日（デフォルト: 今日 $CURRENT_DATE）

## ステップ2: 文字起こしを受け取る

ユーザーに「HiNotesの文字起こしテキストを貼り付けてください」と依頼。
全文を受け取ったら、以下のステップに進む。

## ステップ3: ナレッジファイルを読み込む

以下の3ファイルを Read ツールで読み込んでから分析に使用すること:

1. `ナレッジ格納庫/SPIN型面談スクリプト.md` — SPIN各Phaseの評価基準
2. `ナレッジ格納庫/面談後フォロー3-7-14ルール.md` — LINEテンプレート
3. `ナレッジ格納庫/個別相談セールス分析レポート.md` — 過去12件の比較パターン

## ステップ4: 分析レポート生成

文字起こしから以下を抽出・分析し、mdファイルとして保存する。

**保存先:** `ナレッジ格納庫/面談分析/{面談日}_{名前}.md`

**フォーマット:**

```
# 面談分析レポート: {名前}

## 基本情報
- 面談日: {日付}
- 名前: {名前}
- 学年: {学年}
- 同席者: {同席者}
- 面談時間: {推定分数}分（文字起こしのタイムスタンプから推定）

## 自動抽出項目
- 痛み（現状の不満）: {文字起こしから抽出}
- 理想（将来像）: {文字起こしから抽出}
- 障壁（決断を妨げる要因）: {文字起こしから抽出}
- 温度感: {高/中/低}
- 意思決定者: {本人/母/父/その他}

## SPIN評価スコアカード
| Phase | 内容 | 評価（5段階） | コメント |
|-------|------|:---:|----------|
| Phase 1 | アイスブレイク＋状況確認 | {1-5} | {評価理由} |
| Phase 2 | 痛みの深掘り | {1-5} | {評価理由} |
| Phase 3 | 理想の未来を描く | {1-5} | {評価理由} |
| Phase 4 | ギャップの確認 | {1-5} | {評価理由} |
| Phase 5 | サポート提案 | {1-5} | {評価理由} |
| Phase 6 | クロージング | {1-5} | {評価理由} |

## よかった点（3つ）
1. {具体的な発言や対応を引用}
2. {具体的な発言や対応を引用}
3. {具体的な発言や対応を引用}

## 問題点（3つ）
1. {具体的な問題と、なぜ問題かの説明}
2. {具体的な問題と、なぜ問題かの説明}
3. {具体的な問題と、なぜ問題かの説明}

## 改善アクション（3つ）
1. {次回から実行できる具体的なアクション}
2. {次回から実行できる具体的なアクション}
3. {次回から実行できる具体的なアクション}

## 過去データとの比較
- 成約者の平均面談時間（40分）との差: {差分}
- 最も近い過去パターン: {過去12件から最も類似するケース名}
- 類似点と相違点: {簡潔に}
```

## ステップ5: LINE文面4通生成

`面談後フォロー3-7-14ルール.md` のテンプレートを参照し、面談内容でカスタマイズしたLINE文面を生成する。

**保存先:** `ナレッジ格納庫/LINE下書き/{面談日}_{名前}.md`

**フォーマット:**

```
# LINE下書き: {名前}（{面談日}面談）

---

## 【当日】お礼LINE
送信タイミング: 面談終了後30分以内

{テンプレートA〜Dから温度感に合うものを選び、面談内容でカスタマイズ}

---

## 【3日後】フォローLINE
送信日: {面談日+3日}

{テンプレートE〜Gから状況に合うものを選び、カスタマイズ}

---

## 【7日後】期限通知LINE
送信日: {面談日+7日}

{テンプレートH〜Jから状況に合うものを選び、カスタマイズ}

---

## 【14日後】最終確認LINE
送信日: {面談日+14日}

{テンプレートK〜Lから状況に合うものを選び、カスタマイズ}
```

**選択基準:**
- 温度感「高」（仮申込済み）→ テンプレート A, E, H, K
- 温度感「中」（検討中）→ テンプレート B, F, I, L
- 温度感「低」（保護者相談必要）→ テンプレート C or D, G, J, L
- 各テンプレートの {面談要約}, {痛み}, {理想}, {障壁}, {名前} を面談内容で置換

## ステップ6: Asanaタスク作成（ブラウザ+API ハイブリッド方式）

**プロジェクト:** オールインワンプラン契約書（GID: `1209960384497212`）
**セクション:** 見込み客（GID: `1209960384497232`）

> ⚠️ `create_task_preview` ウィジェットは Claude Code 環境で描画されないため、
> Edgeブラウザ操作 + Asana MCP API のハイブリッド方式を使用する。

### 6-1. メインタスク作成（Edgeブラウザ）
1. `tabs_context_mcp` でEdgeタブを取得
2. Asanaプロジェクトページに移動: `https://app.asana.com/0/1209960384497212/list`
   - デスクトップアプリにリダイレクトされた場合は「ブラウザで開く」リンクをクリック
3. 「見込み客」セクションの「タスクを追加...」をクリック
4. タスク名を入力: `{名前}さん 面談フォロー` → Enter

### 6-2. メインタスクの詳細設定（Asana MCP API）
1. `search_objects` でタスクGIDを取得（タスク名で検索）
2. `update_task` で以下を設定:
   - `due_on`: {面談日+14日}（YYYY-MM-DD形式）
   - `notes`: **面談分析レポート全文を貼り付ける**（ステップ4で生成した内容）
     - 基本情報（名前・学年・同席者・面談時間）
     - 自動抽出項目（痛み・理想・障壁・温度感・意思決定者）
     - SPIN評価スコアカード
     - よかった点（3つ）・問題点（3つ）・改善アクション（3つ）
     - 過去データとの比較

### 6-3. サブタスク4件作成（Edgeブラウザ → API）

**ブラウザでサブタスク追加:**
1. タスク詳細パネルで「＋ サブタスクを追加」をクリック
2. 以下を順にEnterで入力:
   - `【当日】お礼LINE送信`
   - `【3日後】進捗確認LINE`
   - `【7日後】期限通知LINE`
   - `【14日後】最終確認LINE`
3. Escで入力終了

**APIで期限＋LINE文面を設定:**
`search_objects` で各サブタスクのGIDを取得し、`update_task` で期限と説明を一括設定:

| サブタスク名 | 期日 | `notes` に設定する内容 |
|-------------|------|----------------------|
| 【当日】お礼LINE送信 | {面談日} | ステップ5のLINE下書き「当日」セクション全文 |
| 【3日後】進捗確認LINE | {面談日+3日} | ステップ5のLINE下書き「3日後」セクション全文 |
| 【7日後】期限通知LINE | {面談日+7日} | ステップ5のLINE下書き「7日後」セクション全文 |
| 【14日後】最終確認LINE | {面談日+14日} | ステップ5のLINE下書き「14日後」セクション全文 |

> 💡 サブタスクを開けばそのまま LINE にコピペできるよう、送信文面をそのまま `notes` に入れる。

## ステップ7: Googleスプレッドシート更新（ClipboardEvent方式）

Edge ブラウザ MCP を使用して、広告分析ダッシュボード（スプレッドシートID: `1lq2N5KtvdNaFqZweEaN2n3nAoFuHRrWKUU8TIseZN_M`）の「面談DB」シートにデータを追記する。

**操作手順:**
1. `tabs_context_mcp` でEdgeタブを取得
2. スプレッドシートURLに移動: `https://docs.google.com/spreadsheets/d/1lq2N5KtvdNaFqZweEaN2n3nAoFuHRrWKUU8TIseZN_M/edit`
3. 「面談DB」シートタブをクリック（`find` で ref を取得 → `left_click`）
4. Name Box（左上のセル参照欄）をクリックし、空き行のセル番号（例: `A2`）を入力 → Enter
5. `javascript_tool` で ClipboardEvent を使ったペースト:

```javascript
const rowData = "{面談日}\t{名前}\t{属性}\t{同席者}\t{温度感}\t{障壁}\t{面談時間}\t{ステータス}\t{次回アクション日}\t{次回アクション内容}\t\t{備考}";
const dt = new DataTransfer();
dt.setData('text/plain', rowData);
const evt = new ClipboardEvent('paste', { clipboardData: dt, bubbles: true, cancelable: true });
document.activeElement.dispatchEvent(evt);
```

> ⚠️ `computer(type)` はIME変換で不安定。`ClipboardEvent` 方式が最も確実。

**列定義:**

| 列 | 内容 |
|----|------|
| A | 面談日 |
| B | 名前 |
| C | 属性（学年・性別） |
| D | 同席者 |
| E | 温度感（高/中/低） |
| F | 障壁 |
| G | 面談時間（分） |
| H | ステータス（検討中/仮申込/成約/失注） |
| I | 次回アクション日 |
| J | 次回アクション内容 |
| K | 成約日（該当時は空欄） |
| L | 備考 |
