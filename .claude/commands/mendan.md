# /mendan — 面談後フォロー自動化コマンド

面談後の分析・フォロー業務を一括実行します。

## ステップ1: 旧PCから面談データを取得（自動）

まず旧PCのAPIから未処理の面談データを確認する。

**実行方法:** Bash ツールで以下を実行:

```
curl -s http://100.74.20.91:3848/api/mendan/pending
```

### ケースA: 未処理面談がある場合（推奨フロー）

旧PCのHiNote監視が自動検出した面談データを使用する。ブラウザ操作不要。

1. レスポンスのJSON配列から最新のエントリを選択
2. 面談者の名前・日時・文字起こし・UTAGE情報が全て含まれている
3. `POST http://100.74.20.91:3848/api/mendan/{id}/claim` でclaimする
4. ステップ3に直接進む

**取得できるデータ:**
- `id`: 面談エントリID（claim/done APIで使用）
- `name`: 面談者名（タイトルから自動抽出）
- `date`: 面談日
- `transcription`: 文字起こし全文
- `segmentCount`: セグメント数
- `participant`: UTAGE申し込みフォーム情報（自動取得済み）

### ケースB: 未処理面談がない場合（手動フロー）

従来通りユーザーに情報を質問する:
- 面談者の名前（フルネーム）
- 学年（高1/高2/高3/大学生/社会人/保護者）
- 同席者（本人のみ/母同席/父同席/両親同席）
- 面談日（デフォルト: 今日 $CURRENT_DATE）

その後「HiNotesの文字起こしテキストを貼り付けてください」と依頼。

## ステップ2: 複数面談がある場合の選択

APIから複数件の面談が返ってきた場合、一覧を表示してユーザーに選択してもらう:

```
未処理の面談が {N}件 あります:
1. {名前} — {日時}（{segmentCount}セグメント）
2. {名前} — {日時}（{segmentCount}セグメント）

どれを分析しますか？（番号で指定、または「全部」）
```

## ステップ3: ナレッジファイルを読み込む

以下の3ファイルを Read ツールで読み込んでから分析に使用すること:

1. `knowledge/sales/SPIN型面談スクリプト.md` — SPIN各Phaseの評価基準
2. `knowledge/sales/面談後フォロー3-7-14ルール.md` — LINEテンプレート
3. `knowledge/sales/個別相談セールス分析レポート.md` — 過去12件の比較パターン

## ステップ5: LINE文面4通生成

`面談後フォロー3-7-14ルール.md` のテンプレートを参照し、面談内容でカスタマイズしたLINE文面を生成する。

**保存先:** `knowledge/students/LINE下書きまとめ.md` に追記（末尾に `---` 区切りで追加。目次にもリンクを追加）

**フォーマット:**

```
# LINE下書き: {名前}（{面談日}面談）

---

## 【当日】お礼LINE
送信タイミング: 面談終了後30分以内

{テンプレートA〜Dから温度感に合うものを選び、面談内容でカスタマイズ}

---

## 【3日後】フォローLINE
送信日: {面談日+3日}

{テンプレートE〜Gから状況に合うものを選び、カスタマイズ}

---

## 【7日後】期限通知LINE
送信日: {面談日+7日}

{テンプレートH〜Jから状況に合うものを選び、カスタマイズ}

---

## 【14日後】最終確認LINE
送信日: {面談日+14日}

{テンプレートK〜Lから状況に合うものを選び、カスタマイズ}
```

**選択基準:**
- 温度感「高」（仮申込済み）→ テンプレート A, E, H, K
- 温度感「中」（検討中）→ テンプレート B, F, I, L
- 温度感「低」（保護者相談必要）→ テンプレート C or D, G, J, L
- 各テンプレートの {面談要約}, {痛み}, {理想}, {障壁}, {名前} を面談内容で置換

## ステップ7: Asanaタスク作成（MCP API方式）

**プロジェクト:** オールインワンプラン契約書（GID: `1209960384497212`）
**セクション:** 見込み客（GID: `1209960384497232`）

> ブラウザ操作不要。Asana MCPプラグイン（`asana_create_task`）で全て完結する。
> `create_task_preview` はCLIで描画されないため使わない。`asana_create_task` を直接使う。
> **日報プロジェクト（ID: 1209935959800165）のみ**担当者・期日を設定しない。
> **このプロジェクト（オールインワンプラン契約書）ではメインタスク・サブタスク共に担当者・期日を設定する。**

### 7-1. メインタスク作成

`mcp__plugin_asana_asana__asana_create_task` で作成:
- `name`: `{名前}さん 面談フォロー`
- `project_id`: `1209960384497212`
- `section_id`: `1209960384497232`
- `assignee`: `1208498758664385`
- `due_on`: `{面談日+14日}`（最後のサブタスクと同じ日）
- `notes`: 面談分析レポート（ステップ4の内容、**文字起こし全文セクションは除く**）
  - 基本情報（名前・学年・同席者・面談時間）
  - 自動抽出項目（痛み・理想・障壁・温度感・意思決定者）
  - SPIN評価スコアカード
  - よかった点（3つ）・問題点（3つ）・改善アクション（3つ）
  - 過去データとの比較

→ レスポンスからメインタスクの `gid` を取得する。

### 7-2. サブタスク4件作成

`asana_create_task` で4件作成し、`asana_set_parent_for_task` で親子関係を設定:

| サブタスク名 | `due_on` | `notes` に設定する内容 |
|-------------|----------|----------------------|
| 【当日】お礼LINE送信 | {面談日} | ステップ5のLINE下書き「当日」セクション全文 |
| 【3日後】進捗確認LINE | {面談日+3日} | ステップ5のLINE下書き「3日後」セクション全文 |
| 【7日後】期限通知LINE | {面談日+7日} | ステップ5のLINE下書き「7日後」セクション全文 |
| 【14日後】最終確認LINE | {面談日+14日} | ステップ5のLINE下書き「14日後」セクション全文 |

各サブタスクの作成手順:
1. `asana_create_task` で作成（`name`, `assignee`, `due_on`, `notes` を設定）
2. `asana_set_parent_for_task` で `parent_id` にメインタスクの `gid` を指定

> **重要: `notes` にはLINE送信文面の全文をそのまま入れること。概要や要約ではなく、ステップ5で生成した文面をコピペ可能な状態でそのまま入れる。** サブタスクを開く → notesをコピー → LINEに貼り付け、で完結させる。

## ステップ8: Googleスプレッドシート更新（gws CLI方式）

gws CLI の `values append` で「面談DB」シートにデータを追記する。ブラウザ操作不要。

> 日本語JSONはBash経由だとShift-JIS変換で文字化けするため、Node.js経由で実行する。
> `append` はテーブル末尾を自動検出するため、空き行を手動で探す必要なし。

**実行方法:**

Node.js の `require('child_process')` 経由で以下のgwsコマンドを組み立てて実行:

```
gws sheets spreadsheets values append
  --params '{"spreadsheetId":"1lq2N5KtvdNaFqZweEaN2n3nAoFuHRrWKUU8TIseZN_M","range":"面談DB!A:L","valueInputOption":"USER_ENTERED"}'
  --json '{"values":[["{面談日}","{名前}","{属性}","{同席者}","{温度感}","{障壁}","{面談時間}","{ステータス}","{次回アクション日}","{次回アクション内容}","","{備考}"]]}'
```

- paramsとjsonは `JSON.stringify(JSON.stringify(...))` で二重エスケープしてCLIに渡す
- レスポンスの `updatedRange` で書き込み先セルを確認できる

**列定義:**

| 列 | 内容 |
|----|------|
| A | 面談日 |
| B | 名前 |
| C | 属性（学年・性別） |
| D | 同席者 |
| E | 温度感（高/中/低） |
| F | 障壁 |
| G | 面談時間（分） |
| H | ステータス（検討中/仮申込/成約/失注） |
| I | 次回アクション日 |
| J | 次回アクション内容 |
| K | 成約日（該当時は空欄） |
| L | 備考 |
