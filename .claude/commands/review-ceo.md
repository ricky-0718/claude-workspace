# /review-ceo — 経営者視点レビュー

直前の作業内容、または指定された施策・成果物を**経営者の目線**でレビューしてください。

## 判断基準（必ず読み込んでからレビューすること）

**`.claude/skills/ceo-review-criteria.md` を最初に読み込む**こと。このファイルには：
- ROI判定基準（数値ベースの閾値）
- 事業のボトルネック優先順位（実データに基づく7項目）
- おさる式売上方程式での変数チェック
- 現在のフェーズ判定

このSkillsファイルのルールに照合してレビューする。「感覚」や「一般論」での判断は禁止。

## レビューの問いかけ（5つ）

1. **戦略整合性**: ボトルネック優先順位1〜7のどれに対応するか？対応しないなら優先度は低い
2. **ROI判断**: コストゼロの未着手施策が残っている状態で、コストがかかる施策に手を出していないか？
3. **機会損失**: おさる式売上方程式で見て、リスト数とLTVのどちらに効くか？どちらにも効かないなら要注意
4. **スケーラビリティ**: 1回限りか仕組みか？同じ効果をもっと簡単に達成する方法はないか？
5. **リスク**: 失敗した場合のダウンサイドは取り返しがつくか？

## 参照すべきナレッジ

- **`.claude/skills/ceo-review-criteria.md`** — 判断基準（必須・最初に読む）
- `knowledge/business/事業戦略ロードマップ.md` — 施策一覧と優先順位
- `knowledge/business/統合提案書.md` — 事業全体の分析
- `knowledge/marketing/osaru-methodology.md` — おさる式メソドロジー

## 出力フォーマット

```
━━━ CEO Review ━━━

■ 対象: [レビュー対象の施策・成果物]

■ 戦略整合性: [○ 合致 / △ 部分的 / × 不整合]
  [理由を1-2文で]

■ ROI判断: [○ 高い / △ 普通 / × 低い]
  [投下時間 vs 期待リターンの評価]

■ 機会損失:
  [今やるべきだった別の選択肢があれば指摘]

■ スケーラビリティ: [仕組み化されている / 1回限り]

■ リスク:
  [見落としているリスクがあれば指摘]

■ 経営者としての判断:
  [続行 / 方向修正 / 中止 のいずれかと、その理由]
```

## 注意

- お世辞は不要。「良くできました」ではなく「やるべきだったか」を評価する
- 事業戦略ロードマップの優先順位と照合すること
- 短期的な「便利さ」より長期的な「事業インパクト」で判断する
