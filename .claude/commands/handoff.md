# セッション引き継ぎ（ハンドオフ）

現セッションの作業内容を整理し、次セッションがスムーズに再開できる引き継ぎドキュメントを作成する。

## ワークフロー

### Step 1: 現セッションの作業内容を収集
1. `git diff --stat HEAD~5..HEAD`（直近のコミット差分）を確認
2. `git log --oneline -10` で今回のコミット履歴を確認
3. 会話の中で実施した作業を振り返る

### Step 2: 引き継ぎ内容を構成
以下の形式で整理する：

```
# セッション引き継ぎ — [日付] [テーマ]

## 完了したこと
- [具体的な完了事項を箇条書き]

## 未完了・次にやること
- [残タスクを箇条書き。優先度順]

## 重要なコンテキスト
- [次セッションで知っておくべき背景情報]
- [判断の経緯、なぜこのアプローチを選んだか]
- [エラーや注意点があれば記載]

## 関連ファイル
- [変更・作成したファイルパス一覧]
```

### Step 3: ファイルに保存
1. `tasks/handoff-[テーマ].md` に保存（テーマは英語のケバブケース）
2. 同名ファイルが既にある場合は上書き（最新状態が正）

### Step 4: メモリ更新の判断
1. MEMORY.mdに記録すべき新しい情報があるか確認
2. あれば更新を提案・実行

### Step 5: コミット
1. 引き継ぎドキュメントとメモリ更新をコミット
2. ワークツリーにいる場合はプッシュも実施

## 注意事項
- 技術的な詳細より「次に何をすべきか」を優先して書く
- 未完了タスクには再開に必要な情報（URL、ID、コマンド等）を含める
- セッション固有の一時情報（デバッグ途中の仮説など）も残す
