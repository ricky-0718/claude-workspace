# /team-audit — チーム整合性チェック

以下の3つの検証を実行し、問題があれば修正案を提示してください。

## 検証1: エージェント ↔ CLAUDE.md ルーティング表

1. `.claude/agents/` 内の全 `.md` ファイルを列挙する
2. `CLAUDE.md` の「仮想マーケティングチーム > エージェント一覧」テーブルと照合する
3. **不一致を検出**: agents/ にあるがCLAUDE.mdにない、またはその逆

## 検証2: エージェント → knowledge 参照パスの存在確認

1. 各エージェントファイルの「参照すべきナレッジ」セクションからファイルパスを抽出する
2. 各パスが実際に存在するか確認する
3. **壊れた参照を検出**: 参照先ファイルが存在しない

## 検証3: knowledge → エージェント 逆引き（死蔵ファイル検出）

1. `knowledge/` 内の全 `.md` ファイルを列挙する（README.md は除外）
2. いずれかのエージェントから参照されているか確認する
3. **死蔵ナレッジを検出**: どのエージェントからも参照されていないファイル
   ※ 新しく追加されたばかりのファイルは死蔵ではなく「未割り当て」として報告

## 出力フォーマット

```
━━━ チーム整合性レポート ━━━

■ エージェント登録状況
  agents/ のファイル数: X
  CLAUDE.md 登録数: Y
  → [一致 / 不一致あり（詳細）]

■ 参照パス検証
  総参照数: X
  存在: Y / 欠落: Z
  → [全件正常 / 欠落あり（詳細）]

■ ナレッジ活用状況
  knowledge/ のファイル数: X（README除外）
  参照済み: Y / 未割り当て: Z
  → [未割り当てファイル一覧]

■ 推奨アクション
  1. [具体的な修正内容]
  2. [具体的な修正内容]
```

問題が見つかった場合は、修正を実行するか確認してください。
