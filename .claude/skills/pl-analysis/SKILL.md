## ワークフロー

### Phase 1: 情報収集

1. **対象コホートの確認**
   - 何期生か、プロダクト構成（AIO/サマキャン/春キャン/他）
   - 生徒数・単価・班構成
   - 前期（4期生等）からの変更点

2. **授業スケジュールの確認**
   - 課程總覽スプレッドシートから各班の授業時間を集計
   - 講師ごとの単価を確認（涵涵先生¥2,500/h、TOEIC¥5,000/h等）

3. **Freeeから費用データを取得**
   - 損益計算書（/reports/profit_and_loss）で全科目の月次データ
   - 総勘定元帳で各科目の取引明細（サービス別集計）
   - **二重計上チェック**: プロダクトP&Lに既に含まれている費用を除外
     - ASANO TRAVEL旅費 → 住宿費と二重
     - 教科書代に含まれるもの → 消耗品費・研修費と二重
     - UTAGE → 交際費に計上ミスされている場合あり
     - いいオフィス → 会議費と地代家賃に分散
     - 税理士 → 支払手数料と支払報酬料に分散

### Phase 2: スプレッドシート構築

4. **新規スプレッドシート作成**（または既存をコピー）

5. **各プロダクトP&Lタブを作成**
   - セクション順序: 収入 → コスト → 損益 → 1人あたり分析 → 損益合計 → BEP → **シミュレーション（最下部）**
   - 全セルを数式で（ハードコード禁止）
   - シミュレーション: 5名刻み、5〜40名

6. **総合P&Lタブを作成**
   - プロダクト別貢献利益（クロスタブ参照）
   - 会社固定費: 人件費 → 管理費 → その他経費 → 合計
   - スタッフの月数が異なる場合は =月額*月数 の数式 + 注釈に「Xヶ月分」
   - 最終損益・BEP・シミュレーション

7. **経営分析レポートタブを作成**（一番左に配置）
   - 総合診断（利益率・固定費比率・BEP余裕）
   - コスト構造分析
   - プロダクト別評価
   - 売上拡大施策
   - 施策別シミュレーション

### Phase 3: デザイン適用

8. **プロレベルのデザイン**
   - カラーパレット: navy/steel/tblHead/subtotal/alt/gold/green/redBg
   - セクション見出し: steel背景 + 白文字 + bold + 11pt
   - 小計行: subtotal背景 + bold + 上下ボーダー
   - 赤字行: redBg + redTxt
   - 現在値: gold背景
   - 数値フォーマット: #,##0 / 0.0%
   - **背景色汚染に注意**: fieldsマスクで変更対象のみ指定

### Phase 4: 検証

9. **整合性チェック**
   - 各タブの売上合計 = 総合P&Lのプロダクト合計（クロスチェック）
   - 各タブの利益合計 = 総合P&Lの貢献利益合計
   - エラーセル（#DIV/0! #REF!等）のスキャン
   - シミュレーションの現在値行が実績と一致

10. **ユーザーと確認**
    - 各コスト項目の妥当性を確認
    - 二重計上がないか確認
    - スタッフの月数・金額を確認

### Phase 5: 経費内訳分析（オプション）

11. **Freee取引明細のサービス別集計**
    - サブエージェントに委任可能
    - 各勘定科目の総勘定元帳を取得
    - 摘要テキストからサービス名を抽出・グルーピング
    - 月平均・年額換算を算出
