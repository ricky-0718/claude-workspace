# 面談後フォロー自動化

面談後の分析・フォロー業務を一括実行する。

## Gotchas

技術的な注意点・ハマりポイントは `gotchas.md` を参照。

## ステップ1: 旧PCから面談データを取得（自動）

まず旧PCのAPIから未処理の面談データを確認する。

```
curl -s http://100.74.20.91:3848/api/mendan/pending
```

### ケースA: 未処理面談がある場合（推奨フロー）

旧PCのHiNote監視が自動検出した面談データを使用する。ブラウザ操作不要。

1. レスポンスのJSON配列から最新のエントリを選択
2. `POST http://100.74.20.91:3848/api/mendan/{id}/claim` でclaimする
3. ステップ3に直接進む

**取得できるデータ:** id, name, date, transcription, segmentCount, participant

### ケースB: 未処理面談がない場合（手動フロー）

ユーザーに情報を質問: 名前、学年、同席者、面談日。
その後「HiNotesの文字起こしテキストを貼り付けてください」と依頼。

## ステップ2: 複数面談がある場合の選択

APIから複数件返ってきた場合、一覧を表示してユーザーに選択してもらう。

## ステップ3: ナレッジファイルを読み込む

以下の3ファイルを Read ツールで読み込んでから分析に使用:

1. `knowledge/sales/SPIN型面談スクリプト.md`
2. `knowledge/sales/面談後フォロー3-7-14ルール.md`
3. `knowledge/sales/個別相談セールス分析レポート.md`

## ステップ4: 分析レポート生成

文字起こしから抽出・分析し、mdファイルとして保存する。

**保存先:** `knowledge/students/面談分析まとめ.md` に追記（末尾に `---` 区切り）

**フォーマット:** `templates/report-format.md` を参照

## ステップ5: LINE文面4通生成

`面談後フォロー3-7-14ルール.md` のテンプレートを参照し、面談内容でカスタマイズしたLINE文面を生成する。

**保存先:** `knowledge/students/LINE下書きまとめ.md` に追記（末尾に `---` 区切り）

**フォーマット:** `templates/line-draft-format.md` を参照

## ステップ6: 顧客コンテキスト更新

`knowledge/students/顧客コンテキスト.md` に顧客カードを追加する。

**追加先:** ステータスに応じて「成約済み」または「検討中」セクションの末尾に追記。

**フォーマット:**
```
### {名前}（{よみがな}）
- **属性**: {学年} | {同席者} | 面談日 {M/DD}
- **流入**: {どこで知ったか}
- **温度感**: {高/中/低}（{根拠}）
- **意思決定者**: {誰が最終判断するか}
- **志望分野**: {志望学科・将来やりたいこと}
- **主な障壁**: {決断を妨げている要因}
- **ステータス**: {申込済み/検討中/等}
- **フォロー**: 当日{M/DD} → 3日後{M/DD} → 7日後{M/DD} → 14日後{M/DD}
- **詳細**: [面談分析](面談分析まとめ.md#{名前}) | [LINE下書き](LINE下書きまとめ.md#{名前})
```

## ステップ7: Asanaタスク作成（MCP API方式）

**プロジェクト:** オールインワンプラン契約書（GID: `1209960384497212`）
**セクション:** 見込み客（GID: `1209960384497232`）

> `asana_create_task` で直接作成。`create_task_preview` は使わない。
> このプロジェクトではメインタスク・サブタスク共に担当者・期日を設定する。

### 7-1. メインタスク
- `name`: `{名前}さん 面談フォロー`
- `assignee`: `1208498758664385`
- `due_on`: `{面談日+14日}`
- `notes`: 分析レポート（文字起こし全文セクションは除く）

### 7-2. サブタスク4件

| サブタスク名 | `due_on` | `notes` |
|-------------|----------|---------|
| 【当日】お礼LINE送信 | {面談日} | LINE下書き「当日」全文 |
| 【3日後】進捗確認LINE | {面談日+3日} | LINE下書き「3日後」全文 |
| 【7日後】期限通知LINE | {面談日+7日} | LINE下書き「7日後」全文 |
| 【14日後】最終確認LINE | {面談日+14日} | LINE下書き「14日後」全文 |

> **重要:** notesにはLINE文面の全文をコピペ可能な状態でそのまま入れる。

## ステップ8: Googleスプレッドシート更新

gws CLI の `values append` で「面談DB」シートにデータを追記する。
日本語JSONはNode.js `execSync` 経由で実行（Bashからは文字化け）。

## ステップ9: 完了通知・報告

### 9-1. 旧PCに完了通知（ケースAの場合のみ）
```
curl -s -X POST http://100.74.20.91:3848/api/mendan/{id}/done
```

### 9-2. 完了報告
```
面談分析完了: {名前}さん（{日付}）
分析レポート: knowledge/students/面談分析まとめ.md に追記
LINE下書き: knowledge/students/LINE下書きまとめ.md に追記
顧客カード: knowledge/students/顧客コンテキスト.md に追加
Asanaタスク: メイン1件 + サブタスク4件作成済み
スプレッドシート: 面談DB更新済み
温度感: {高/中/低}
次のアクション: {当日LINEを送信}
```
