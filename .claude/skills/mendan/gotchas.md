# mendan Gotchas

面談後フォロー自動化で蓄積されたハマりポイントと対処法。

## 旧PC API関連

- **接続できない場合** → 旧PCがスリープ or Tailscale切断。ユーザーに確認
- **participant情報がない** → UTAGE申し込みフォームが未登録。手動でUTAGEを確認
- **claimし忘れ** → 他セッションと競合する。必ず最初にclaimする

## 文字起こし関連

- **途中で切れている場合がある** → セグメント数を確認し、短すぎる場合はユーザーに再取得を依頼
- **面談時間の推定** → タイムスタンプの最初と最後の差分で計算
- **話者の区別** → HiNoteは話者ラベルが不安定。文脈から判断する

## Circleback連携関連

- **名前が"Participant 1"になる** → Circlebackはwebhookで外部参加者を匿名化する場合がある。3段階で解決: ①スプレッドシート（面談DB）照合 → ②Claude分析レポートから抽出 → ③Circlebackメタデータ（フォールバック）（3/22修正済み）
- **日付が1日ずれる** → `payload.createdAt` はUTCタイムスタンプ。JST変換せずに `.getDate()` するとUTC日付になり1日前になる。`extractMeetingData()` でJST明示変換する（3/22修正済み）
- **LINE名をAsanaに自動設定** → `spreadsheet-lookup.js` で面談DBからLINE名を取得し、`asana-mendan.js` がカスタムフィールド（GID: `1209960385083651`）に自動設定（3/22追加）
- **スプレッドシートの同日複数面談** → 現在は最初のマッチを使用。将来的に時間帯マッチングで改善可能

## Asana関連

- **project_id**: `1209960384497212`（オールインワンプラン契約書）
- **section_id**: `1209960384497232`（見込み客）
- **assignee, due_on は必ず設定する**（日報プロジェクトとは異なるルール）
- **`create_task_preview` は使わない** → `asana_create_task` を直接使う
- **サブタスクのnotes** → LINE文面の全文をそのまま入れる（要約ではなくコピペ可能な状態）

## スプレッドシート関連

- **面談DBへの書き込み** → Node.js execSync経由（Bashから日本語JSONは文字化け）
- **`append` API** → テーブル末尾を自動検出するため、空き行を手動で探す必要なし
- **二重エスケープ** → `JSON.stringify(JSON.stringify(...))` でCLIに渡す

## LINE文面関連

- **温度感の判定が最重要** → 高/中/低で選ぶテンプレートが全く変わる
- **面談で提案しなかった選択肢** → 新しい提案として盛り込む
- **「素晴らしいと思います」は禁止** → 「嬉しいです」に置換（上から評価 → 同じ目線で共感）
- **「！」は多用しない** → 要所で1つ入れると感情が伝わる
