### ボトルネック判定ルール

施策を評価するとき、ファネルの**最も弱いステップ**から改善するのが鉄則。

2. **成約率**（42%→70%で全体1.7倍）← 面談改善で対応可能

### LP改善の判定基準

- ファーストビューの離脱が80% → **ファーストビュー以外の改善はROIが低い**
- スクロール深度13% → ページ下部の改善は効果が薄い（そこまで見られていない）
- 28秒しか滞在しない → **3秒で「自分向けだ」と伝わらなければ離脱する**

## A/Bテスト判定基準

| 項目 | 基準 |
|---|---|
| テスト期間 | 最低7日間（曜日の偏りを排除） |
| 最低サンプル数 | 各バリアント100UU以上 |
| 有意差判定 | 登録率の差が1.5倍以上、かつ3日連続で同じ傾向 |
| テスト変数 | 1回のテストで変えるのは1要素だけ |

## チャネル別の判定基準

### Meta広告
- CPAが上昇傾向 → クリエイティブ疲弊を疑う（同じ広告3週間以上は要交換）
- CTRが下がらずCVRが下がる → LP側の問題（広告とLPのメッセージ不一致）
- CTRもCVRも下がる → ターゲティングの問題

### LINE
- 配信枠30,000通中ほぼ未使用（利用率0.58%）
- LINEを使った施策は追加コストゼロ → 常に検討すべき
- リマインド配信はウェビナー視聴率に直結

## レビュー時の必須チェック

1. この施策はファネルのどのステップに効くか？
2. そのステップは現在ボトルネックか？
3. ボトルネックでないステップへの施策なら、なぜ今やるのか明確な理由があるか？
4. KPIは設定されているか？「なんとなく良くなった」で終わらないか？
5. 測定方法は確立されているか？（Clarity, GA4, UTAGE管理画面）
