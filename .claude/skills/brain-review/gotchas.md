# Brain Review — Gotchas

## Asana API

- `get_my_tasks` は claude.ai Asana MCP と plugin Asana MCP の両方で利用可能。接続中の方を使う
- `completed_since: "now"` で未完了のみ取得。`"now"` は文字列で渡す
- assignee_section.name でセクション名が取れる。セクション移動は現状APIサポートが限定的
- タスクの一括更新はAPIの並列呼び出しで対応（Asanaにはbatch APIがない）

## 他プロジェクトから来たタスクは名前を変えない

マイタスクには「自分で作ったタスク」と「他のプロジェクトからassigneeとして割り当てられたタスク」の2種類が混在する。
後者のタスク名を変更すると、**元のプロジェクト側のタスク名も変わってしまう**。

**判別方法**: `projects` フィールドが空でない（= どこかのプロジェクトに属している）タスクは名前を変えない。
- `projects: []` → マイタスクで作成。名前変更OK
- `projects: [{name: "オールインワンプラン契約書"}]` → 他PJから来たタスク。名前変更NG

**2026-04-03に実際にやらかした**: 儀保翔音さんのタスク名を書き換えてしまい、契約書PJ側にも影響。即座に元に戻した。

## 判断の注意点

- memory/のプロジェクト記録は「その時点の状態」であり、現在の状態とは異なる可能性がある
- タスクの完了判定は必ずユーザーに確認する。勝手に閉じない
- 「卒業ルール」適用時、ユーザーが次のアクションを言語化できない場合がある。その場合は「このタスクについて、今一番気になっていることは何ですか？」と聞く
- 旧PCのサーバーバグ（findstr "200"ウィンドウ増殖）により空タスクが生成されることがある。名前なし・画像のみのタスクは要確認

## 初回棚卸し（2026-04-03）で判明したパターン

- 「今日の作業」にdue_onなしのタスクが大量に溜まる傾向
- タスク名がプロジェクト名（名詞だけ）になりがち
- リサーチ系タスクは「リサーチ完了→次のアクション未定義」で放置される
- 重複は「戦略メモ」と「実装タスク」が別々に存在するパターンが多い
- 人に関するタスク（面談フォロー等）はAsana契約書PJとマイタスクの二重管理になりやすい
