# 脳の棚卸し（Brain Review）

Asanaマイタスクを「脳のダッシュボード」として機能させるための定期メンテナンス。

## Gotchas

技術的な注意点は `gotchas.md` を参照。

## 実行ログ

過去の棚卸し履歴は `execution-log.md` を参照。パターンの蓄積に使う。

---

## ステップ1: 全タスク取得

Asana MCPプラグインでマイタスクを全件取得する。

```
mcp__plugin_asana_asana__asana_get_my_tasks（claude.ai Asana MCPが接続中の場合）
or
mcp__claude_ai_Asana__get_my_tasks（プラグイン版）
```

パラメータ:
- `completed_since: "now"` — 未完了のみ
- `opt_fields: "name,due_on,completed,assignee_section,assignee_section.name,notes,tags,tags.name,projects,projects.name,memberships,memberships.section,memberships.section.name"`

## ステップ2: 5つの診断を実行

取得した全タスクに対して以下の5チェックを回す。結果をテーブルで表示する。

### 診断A: ゴミ検出
以下に該当するタスクを特定する:
- **タスク名が空** — 名前なしのメモ・画像のみ
- **作成日から30日以上経過 + due_onなし + notesが空** — 放置されたゴースト
- **完了すべき兆候** — 「〜でやめる」「〜は終わった」等のキーワード
- **ブックマーク的タスク** — URLだけ貼ってあるもの

→ ユーザーに「完了にしますか？」と確認

### 診断B: 重複検出
- タスク名の類似度が高いペアを検出（同じキーワードが2つ以上含まれる）
- 同じプロジェクトに属するタスクで内容が重複しているもの
- memory/のプロジェクト記録と照合して「これは完了済みでは？」を検出

→ ユーザーに「統合しますか？」と確認

### 診断C: 「次のアクション」チェック
GTDの原則に基づき、全タスク名を評価:
- **OK**: 動詞で始まる（「〜を送る」「〜を決める」「〜を作る」）
- **NG**: 名詞だけ（「HP修正」「YouTube企画案」「今後の宿舎戦略」）
- **NG**: プロジェクト名になっている（「ガイドブックファネルの再構築」）

NGタスクに対して「次の物理的アクション」への変換案を提示。
タスク名を見て5秒で何をすべきかわからなければNG。

### 診断D: 期日チェック
- **期限切れ**: due_on < 今日 → 延長 or 完了の判断を促す
- **期限なし**: due_on == null → 期日の提案
- **「今日の作業」に期限なしが5件以上**: 認知負荷オーバーの警告

### 診断E: 卒業ルール適用候補
以下のパターンに該当するタスクを検出:
- 「〜をリサーチ」「〜を調べる」系で、memory/に完了記録がある
- notesにリンクや詳細な結果が書かれているが、次のアクションが不明確
- サブタスクが全て完了しているのに親が未完了

→ 「元タスクを完了にして、新タスクを作りますか？」と確認。
→ ユーザーに「このタスクから生まれた次のアクションは何ですか？」と聞く。

## ステップ3: 変更の実行

ユーザーの承認を得た変更を一括実行:
1. 完了にするタスク → `asana_update_task(completed: true)`
2. 名前変換 → `asana_update_task(name: "新しい名前")`
3. 期日設定 → `asana_update_task(due_on: "YYYY-MM-DD")`
4. 卒業ルール → 旧タスク完了 + 新タスク作成（`asana_create_task`）
5. notes更新（統合時のリンク転記等）

## 運用ガイドライン

- **頻度**: 週1回（月曜朝 or 金曜夕方）が理想。最低でも月2回。
- **所要時間**: 15分以内を目標。診断は自動、判断はユーザー。
- **原則**: ユーザーの承認なしにタスクを完了・削除しない。提案→承認→実行の3ステップ。
- **覚えておくべき判断基準**:
  - knowledge/students/の情報とAsana契約書PJが正（面談メモではない）
  - 「もやもや」は言語化を手伝う。「何が残ってますか？」と聞く
  - 5件以上の🔥は認知負荷オーバー。優先順位を絞る
