# utage-check Gotchas

よくあるハマりポイントと対処法。実運用で蓄積されたものです。

## ブラウザ関連

- **Edge専用。Chrome禁止。`switch_browser` 禁止。** CDPモードでEdgeに接続する。Chromeは使えない
- **Edge拡張切断**: 頻繁に起きる。`tabs_context_mcp` で再接続すれば復帰する
- **ページ読み込み待ち**: UTAGEは重いので、navigate後にsnapshotが安定するまで待つ

## UTAGE操作関連

- **2つ目のアカウント「休止」は無視**: チャットリストに表示されるが対応不要
- **メモ更新ボタンの押し忘れ**: メモ入力後は必ず「更新」ボタンを押す。`find` で「更新」ボタンを検索→`ref`でクリック。押さないと保存されない
- **チャットの既読/未読**: チャットを開くだけで既読になる。スキャン時は素早く確認してリストを作ること

## LINE返信品質関連

- **LINE返信ルール**: `knowledge/operations/utage-line-rules.md` を必ず読んでから返信案を作成
- **フィードバック履歴**: `knowledge/operations/utage-line-feedback.md` も参照し、過去の学びを反映
- **語尾「ね」禁止**: ビジネス調でもタメ口調でも、語尾に「ね」は絶対に使わない
- **前回メッセージの繰り返し禁止**: 直前に送った内容と同じ情報を再送しない
- **「送信しました」表現禁止**: あくまで「返信案」の提示。送信はオーナーが判断する

## 深掘り調査関連

- **Asana検索**: LINE名とAsana上の名前が異なる場合がある。スラッシュ区切り（例: 太郎/花子）は親子別アカウント
- **面談分析まとめ**: 分析レポートだけでなく文字起こし全文まで読む。表面的な情報で返信を書かない
- **推測で終わらせない**: サブタスク・コメント・添付まで深掘りする
