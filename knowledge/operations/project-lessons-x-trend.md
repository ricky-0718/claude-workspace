# X AI×マーケ トレンド収集システム — プロジェクトの学び

## 1. GASエディタへのコード反映（ブラウザ自動化）

**確立した最安定パターン: base64チャンク方式**
1. Node.jsでファイル → base64エンコード → 3000文字ずつ分割
2. `window._b64 = '...'` → `window._b64 += '...'` で蓄積
3. `Uint8Array.from(atob(window._b64), c => c.charCodeAt(0))` → `TextDecoder('utf-8')` でデコード
4. `monaco.editor.getModels()[0].setValue(decoded)` で一括反映

他の方式（直接setValue、文字列チャンク）はエスケープ問題で不安定。

## 2. X API v2 制約（Pay Per Use）
- `min_faves` オペレーター使用不可 → GAS側で `MIN_LIKES` フィルタ実装
- Bearer Token「コピー」ボタンがURLエンコード含む場合あり
- `/2/users/me` はapp-only authで動かない（OAuth 2.0必要）
- Token再生成で旧トークン即無効化

## 3. 複数API連携パイプライン設計
- 段階的テスト: 各API個別テスト → 結合テスト
- APIキーはスクリプトプロパティに格納（コード埋め込み禁止）
- 処理済みIDをスプレッドシートで管理し冪等性確保

## 4. パーソナライズ手法
- Xブックマーク分析 → 好みプロファイル作成 → クエリ・フィルタに反映
- プロファイルを先に作ると後のチューニングが楽

## 5. セッション引き継ぎ
- memoryファイルにプロジェクト全情報集約（リソースID、テスト結果、次やること）
- ローカルコードは必ずコミット済みにする
- **反省**: GASエディタとローカルの同期状態を明記すべき

## 6. 再利用パターン
- base64方式はGASコード反映の標準テンプレートにする
- 「API連携→加工→通知」パターンは他の自動化タスクでも同じ構造
- ブックマーク分析→パーソナライズはLP制作・広告コピーにも応用可
