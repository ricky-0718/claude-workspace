# おさるAI GPTs × Claude Code 連携ワークフロー

作成日: 2026-03-21

---

## コア思想: Claude批判的レビュー → おさるAI再生成ループ

おさるAI GPTsの出力をそのまま使わない。
Claude Codeが知識ベース（knowledge/）を持っているからこそ、
**GPTの出力を事業データと照合して批判的にレビューし、改善要求を送り、完璧になるまでループする。**

```
knowledge/から情報収集
    ↓
おさるAI GPTに構造化入力を送信
    ↓
第1版の出力を取得
    ↓
┌─→ Claudeが批判的レビュー（knowledge/と照合）
│       ↓
│   改善ポイントを具体的に指摘
│       ↓
│   おさるAI GPTに改善要求を送信
│       ↓
│   改善版を取得
│       ↓
└── まだ問題あり？ → ループ
        ↓ OK
    成果物をローカル保存（MD + HTML）
    git commit
```

## 5ステップ手順

### Step 1: GPT選定
ユーザーのURLまたはGPT名から特定。全56種のリンクは `osaru-funnel-templates.md` セクション3に記載。

### Step 2: knowledge/から事業データ収集
GPTに送る前に必ず収集:
- `business/会社全事業データ総覧.md` — 会社概要・実績・数字
- `marketing/パーソナルナレッジ.txt` — 代表プロフィール・ペルソナ・口調
- `sales/` — 面談データ・セールス分析・断り文句
- `marketing/osaru-methodology.md` — おさる式理論
- タスクに応じた専門ナレッジ

### Step 3: 構造化入力を送信（Playwright MCP経由）
- GPTが求めるフォーマットに合わせて整理
- 具体的な数字を入れる（「合格率100%」「12名全員合格」等）
- 断り文句は面談分析レポートの実データから
- ターゲットの悩みは生の声（FAQ・面談履歴）から引用

### Step 4: 批判的レビュー＆ブラッシュアップ（最重要）

| チェック観点 | 例 |
|---|---|
| 質問が決めつけ型になっていないか | 「ありますよね」→ オープンクエスチョンに |
| 具体エピソードが薄くないか | 「大失敗した」→ 「8単位しか取れず絶望」に |
| 意思決定者への配慮が足りるか | 保護者スライド1枚 → 3箇所以上に |
| knowledgeの情報と整合しているか | SPIN型スクリプトの構造と一致するか |
| 温度感・口調がリッキーさんに合っているか | パーソナルナレッジの口調と照合 |

改善要求は**具体的に**送る:
1. 何が問題か（スライドX〜Yの質問が決めつけ型）
2. なぜ問題か（相手の本音が引き出せない）
3. どう改善するか（オープンクエスチョンに変更。例：「〇〇」）

### Step 5: 成果物の保存・活用
- テキスト版: knowledge/ 配下にMarkdownで保存
- HTMLプレゼン版: 同ディレクトリにHTMLで保存
- git commit で記録

## Playwright MCP操作メモ

### ChatGPTテキスト入力
- `#prompt-textarea` にフォーカス
- DOM操作で段落要素としてテキストをセット
- input イベントを発火
- `button[data-testid="send-button"]` で送信

### 応答待ち
- 「思考中」テキストが消えるまで待つ（3〜5分かかることあり）
- 途切れたら「スライドXから続けて」で続行依頼

### 応答取得
- `[data-message-author-role="assistant"]` で全アシスタントメッセージ取得
- `.markdown` 要素の innerText を取得
- 複数メッセージに分かれることがある → 全結合
