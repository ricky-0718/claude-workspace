## 1. 記事制作パイプライン

### Phase 1: ブリーフ作成（blog-strategist）

**入力**: 記事番号（blog-100-article-map.mdから）
**処理**:
1. `osaru-methodology.md`の売上方程式（リスト数 x CVR x LTV）で記事位置づけ評価
2. `osaru-funnel-templates.md`でファネル上の位置を決定
3. `/blog-brief`スキルで構造化ブリーフ生成
4. おさる式7ステップへのマッピング案追記
5. 心理トリガー（8つから選定）指定
6. CTA設計（LINE登録先の特典番号を明記）
7. 内部リンク先（同クラスター内3本以上 + ピラーへの戻り）を指定

**出力**: コンテンツブリーフ（KW、構成案、心理トリガー、CTA設計、内部リンク指示）

### Phase 2: 記事執筆（blog-writer）

**入力**: Phase 1のブリーフ
**処理**:
1. `voice-guide-ricky-blog.md`必読（文体DNA）
2. `/blog-outline`でSERP分析ベースの構成作成
3. おさる式7ステップに再構成
4. `writing-rules.md`に完全準拠して執筆
5. 心理トリガーを自然に配置
6. LINE CTA を3箇所に配置（冒頭・中盤・末尾）
7. 内部リンクを記事中に自然に配置

**出力**: 記事本文、メタディスクリプション、タイトルタグ、CTA配置箇所、内部リンク一覧

### Phase 3: 品質管理（blog-editor）

**入力**: Phase 2の記事
**8段階チェック**:
1. `/blog-analyze`で100点スコアリング（5カテゴリ）
2. おさる式7ステップ適合度（○/△/×）
3. 心理トリガー配置検証（最低3つ）
4. Voice Guide一致検証
5. AI臭さ排除チェック
6. LINE CTA配置検証（3箇所 + リードマグネット適合）
7. 内部リンク検証（最低3本 + ピラー戻り）
8. `/blog-seo-check` + `/blog-schema` SEO・スキーマ

## 2. 品質基準

### 品質ゲート（公開判定）

| スコア | 判定 | アクション |
|---|---|---|
| 80点以上 | 公開許可 | そのまま公開 |
| 60〜79点 | リライト | blog-writerに改善指示→再スコアリング |
| 60点未満 | 差し戻し | blog-strategistにブリーフから再設計 |

**スコアに関係なく差し戻し**:
- LINE CTA 3箇所未配置
- 内部リンク3本未満
- Voice Guide Anti-Pattern違反あり
- 「ね」使用あり

### スコア内訳（100点満点）

| カテゴリ | 配点 | 内容 |
|---|---|---|
| コンテンツ品質 | 30点 | 情報の正確性、深さ、独自性 |
| SEO最適化 | 25点 | KW配置、構成、メタ情報 |
| E-E-A-T | 20点 | 経験、専門性、権威性、信頼性 |
| テクニカル | 15点 | 構造化データ、内部リンク、画像ALT |
| AI Citation | 10点 | AI検索エンジンへの最適化 |

### 必須チェック項目
- Voice Guide Anti-Patterns 違反ゼロ
- 「ね」使用ゼロ
- AI臭い語彙ゼロ（「革命的」「画期的」「シームレス」等）
- 心理トリガー3つ以上配置
- おさる式7ステップのうち5ステップ以上適合
- CTA 3箇所配置（冒頭・中盤・末尾）
- 内部リンク3本以上（うちピラーへの戻り1本以上）

## 3. LINE導線設計

### リードマグネット x 記事カテゴリ マッピング

| 記事カテゴリ | メインリードマグネット | サブリードマグネット |
|---|---|---|

| 大学選び | 特典⑤ 大学診断カルテ | 特典② 合格実績データブック |
| 準備・手続き | 特典④ 失敗回避レポート | 特典⑤ 大学診断カルテ |
| 生活 | 特典⑥ 中国語ゼロ→A2ロードマップ | 特典④ 失敗回避レポート |
| 中国語学習 | 特典⑥ 中国語ゼロ→A2ロードマップ | 特典⑦ 親子Q&Aブック |
| 体験談 | 特典④ 失敗回避レポート | 特典② 合格実績データブック |
| エージェント比較 | 特典② 合格実績データブック | 特典⑤ 大学診断カルテ |
| 保護者向け | 特典⑦ 親子Q&Aブック | 特典③ 保護者向け費用・安全 |

### CTA配置ルール（全記事共通）

**配置箇所3箇所**:
1. **記事冒頭（フック直後）**: 軽いバナー型 — 「LINE登録で〇〇を無料プレゼント中」
2. **中盤（課題詳細の後）**: テキスト型 — 記事の流れに溶け込むネイティブCTA
3. **記事末尾（CTA直前）**: ボックス型 — 特典の価値を3-5行で説明+登録ボタン

**CTA文言の原則**:
- 「無料」を明記
- 具体的な数字を入れる（「合格実績データ50校分」等）
- 行動のハードルを下げる（「30秒で受け取れます」等）
- 押し売り禁止（おさる式: 「煽り売りは将来稼げるお金を先に刈り取っている」）

### LINE登録後のフロー
1. 特典DL（信頼構築）
2. 3-7日のステップ配信（既存28通を活用）
3. ウェビナー案内 or 個別相談案内
4. 面談 → 成約
