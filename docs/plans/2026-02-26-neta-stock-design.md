# SNSネタ自動ストックシステム 設計書

**作成日**: 2026-02-26
**ステータス**: 承認済み

## 概要

SNS投稿のネタを自動で収集・生成し、スプレッドシートにストックするシステム。
Google Custom Search APIで検索上位コンテンツを収集し、Claude APIで投稿ネタを生成する。
GAS一本で完結し、サーバー不要で安定稼働する。

## アーキテクチャ

```
[設定シート] テーマ×プラットフォーム一覧
       ↓ (5分おきトリガー)
[GAS] Google Custom Search APIで検索
       ↓
[収集データシート] 検索結果を蓄積
       ↓ (1日1回 朝8時トリガー)
[GAS] Claude APIでネタ生成
       ↓
[ネタストックシート] 投稿ネタ一覧
       ↓ (手動)
[sns-agent-team] ダッシュボードにテーマを入力して原稿生成
```

## 技術的決定事項

| 項目 | 決定 | 理由 |
|---|---|---|
| 収集方法 | Google Custom Search API | Bot ブロック回避。無料100回/日 |
| ネタ生成 | Claude API (GASから直接) | GAS一本完結でシンプル |
| sns-agent-team連携 | 手動コピペ | GASからlocalhost不可。将来自動化可能 |
| 実行制限対策 | ラウンドロビン方式 | GAS 6分制限に対応 |
| APIキー管理 | GASスクリプトプロパティ | コードにハードコードしない |

## スプレッドシート構成

### 設定シート

| 列 | 内容 | 例 |
|---|---|---|
| A: テーマ | 検索テーマ | `台湾留学` |
| B: プラットフォーム | 対象SNS | `instagram` |
| C: 検索クエリ | Custom Search用クエリ | `台湾留学 おすすめ site:instagram.com` |
| D: 最終実行日時 | ラウンドロビン制御用 | `2026/2/26 10:00` |

### 収集データシート

| 列 | 内容 |
|---|---|
| A: 収集日時 | タイムスタンプ |
| B: テーマ | 検索テーマ |
| C: プラットフォーム | SNS名 |
| D: タイトル | 検索結果のタイトル |
| E: URL | リンク |
| F: スニペット | 説明文 |
| G: ネタ生成済み | TRUE/FALSE |

### ネタストックシート

| 列 | 内容 |
|---|---|
| A: 生成日時 | タイムスタンプ |
| B: テーマ | 元テーマ |
| C: プラットフォーム | 対象SNS |
| D: ネタタイトル | 投稿の見出し案 |
| E: 投稿案 | 投稿の本文案 |
| F: 元ネタURL | 参考にした検索結果 |
| G: ステータス | `未使用` / `使用済み` |

## GAS処理フロー

### 関数1: collectContent() — 収集処理
**トリガー**: 5分おき（時間ベーストリガー）

1. 設定シートから「最終実行日時が最も古い行」を1つ選ぶ
2. その行の検索クエリでGoogle Custom Search APIを呼ぶ
3. 結果（最大10件）を収集データシートに追記
   - 重複チェック: URLが既に存在する場合はスキップ
4. 設定シートの「最終実行日時」を更新

### 関数2: generateNeta() — ネタ生成処理
**トリガー**: 1日1回（朝8時）

1. 収集データシートから「ネタ生成済み = FALSE」のデータを取得
2. テーマ×プラットフォームごとにグルーピング
3. 各グループごとにClaude APIを呼ぶ
   - 収集データ（タイトル+スニペット）をコンテキストとして渡す
   - 「このデータを元に、SNS投稿ネタを3〜5個生成して」と指示
4. 生成されたネタをネタストックシートに追記
5. 収集データの「ネタ生成済み」をTRUEに更新

### 関数3: setupTriggers() — 初期セットアップ
手動で1回だけ実行。

1. 既存トリガーを削除（重複防止）
2. collectContent: 5分おきの時間ベーストリガー
3. generateNeta: 毎日8時のトリガー

## Claude APIプロンプト

```
あなたはSNSマーケティングの専門家です。
以下の検索結果データを参考に、「{テーマ}」に関する
{プラットフォーム}向けの投稿ネタを3〜5個生成してください。

各ネタには以下を含めてください：
- title: キャッチーなタイトル（30文字以内）
- body: 投稿本文の案（プラットフォームに適した長さ）
- source_url: 参考にした検索結果のURL

【収集データ】
{収集データをここに展開}

【出力形式】JSON配列
[
  {"title": "...", "body": "...", "source_url": "..."}
]
```

## 事前準備（ユーザー作業）

1. Google Cloud Console でプロジェクト作成 & Custom Search API 有効化
2. APIキー発行
3. Programmable Search Engine 作成（「ウェブ全体を検索」ON）
4. 検索エンジンID取得
5. GASスクリプトプロパティに3つのキーを登録:
   - `CUSTOM_SEARCH_API_KEY`
   - `CUSTOM_SEARCH_ENGINE_ID`
   - `CLAUDE_API_KEY`
