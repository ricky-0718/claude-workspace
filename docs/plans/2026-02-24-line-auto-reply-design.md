# LINE自動返信システム 設計書

**作成日**: 2026-02-24
**ステータス**: 承認済み

## 目的

LINE通知が来るたびに、AIが過去の会話を踏まえてセールス返信ドラフトを自動生成し、Chatworkに蓄積される仕組みを作る。

## 全体フロー

### 自動（24時間稼働）

```
お客さん(LINE) → ウタゲ（受信） → [Chatworkルーム1: 受信通知]
                                          ↓ Chatwork Webhook
                                        GAS（自動処理）
                                          ↓
                                スプレッドシートDBから過去履歴を参照
                                          ↓
                                Claude APIで返信文を生成
                                          ↓
                                [Chatworkルーム2: AIドラフト] に自動投稿
```

### 手動（担当者）

```
担当者がウタゲでお客さんに実際に返信した後
  → 送った文章をコピー
  → [Chatworkルーム3: 送信済みログ] に貼り付け
  → GASが自動でスプレッドシートDBに取り込み
  → AIの返信精度が日々向上
```

### 初回のみ

```
ブラウザでウタゲの過去チャット履歴を全取得
  → スプレッドシートDBに一括保存
  → AIが初日からコンテキストを持った状態でスタート
```

## Chatworkルーム構成

| ルーム | 名前（例） | 投稿者 | 内容 |
|--------|-----------|--------|------|
| ルーム1 | LINE受信通知 | ウタゲ（自動） | お客さんが送ってきたメッセージ |
| ルーム2 | AI返信ドラフト | GAS（自動） | AIが生成した返信候補 |
| ルーム3 | 送信済みログ | 担当者（手動） | 実際にお客さんに送った返信文 |

## スプレッドシートDB構造

| 日時 | 顧客名 | 種別 | メッセージ内容 |
|------|-------|------|--------------|
| 2/24 10:00 | 山田太郎 | 受信 | 先日のセミナーの件で... |
| 2/24 10:01 | 山田太郎 | AIドラフト | 山田さん、ご連絡ありがとう... |
| 2/24 10:30 | 山田太郎 | 送信済み | 山田さん、お世話になっております... |

「送信済み」データが増えるほど、AIの返信品質が向上する。

## 返信文の生成ルール

| 項目 | ルール |
|------|--------|
| 目的 | セールス（購入・申し込みに誘導） |
| トーン | 送信済みテキストの傾向に合わせる |
| 参照 | 過去履歴からお客さんの興味・関心を拾う |
| 長さ | 100〜300文字（LINE返信として自然） |
| 禁止 | 具体的な金額は出さない（相談に誘導） |
| 学習元優先順位 | 送信済みテキスト > 受信メッセージ > AIドラフト |

## 技術構成

| コンポーネント | 種類 | 役割 |
|---------------|------|------|
| GAS（メイン処理） | 新規作成 | Webhook受信→AI生成→Chatwork投稿→DB記録 |
| Chatwork Webhook | 設定 | ルーム1の新メッセージをGASに通知 |
| Chatwork API | 利用 | ルーム2,3のメッセージ読み書き |
| Claude API | 利用 | 返信文の生成 |
| Googleスプレッドシート | 新規作成 | 全履歴DB |
| Edge MCP（初回のみ） | ブラウザ操作 | ウタゲの過去履歴インポート |

## 実装フェーズ

| # | フェーズ | 内容 |
|---|---------|------|
| 0 | 初回インポート | ブラウザでウタゲの過去チャット履歴を全取得→DB保存 |
| 1 | 基盤構築 | スプレッドシートDB作成、Chatworkルーム2,3作成 |
| 2 | 自動ドラフト生成 | GAS + Chatwork Webhook + Claude API構築 |
| 3 | 送信済みログ取り込み | ルーム3→DB自動取り込みのGAS追加 |
