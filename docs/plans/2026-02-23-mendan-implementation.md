**Step 2: mendan.md を作成**

以下の完全な内容で `.claude/commands/mendan.md` を作成する。

````markdown
# /mendan — 面談後フォロー自動化コマンド

面談後の分析・フォロー業務を一括実行します。

## ステップ1: 面談者情報を収集

以下を質問してください（AskUserQuestion推奨）:
- 面談者の名前（フルネーム）
- 学年（高1/高2/高3/大学生/社会人/保護者）
- 同席者（本人のみ/母同席/父同席/両親同席）
- 面談日（デフォルト: 今日 $CURRENT_DATE）

## ステップ2: 文字起こしを受け取る

ユーザーに「HiNotesの文字起こしテキストを貼り付けてください」と依頼。
全文を受け取ったら、以下のステップに進む。

## ステップ3: ナレッジファイルを読み込む

以下の3ファイルを Read ツールで読み込んでから分析に使用すること:

1. `ナレッジ格納庫/SPIN型面談スクリプト.md` — SPIN各Phaseの評価基準
2. `ナレッジ格納庫/面談後フォロー3-7-14ルール.md` — LINEテンプレート
3. `ナレッジ格納庫/個別相談セールス分析レポート.md` — 過去12件の比較パターン

## ステップ4: 分析レポート生成

文字起こしから以下を抽出・分析し、mdファイルとして保存する。

**保存先:** `ナレッジ格納庫/面談分析/{面談日}_{名前}.md`

**フォーマット:**

```
# 面談分析レポート: {名前}

## 基本情報
- 面談日: {日付}
- 名前: {名前}
- 学年: {学年}
- 同席者: {同席者}
- 面談時間: {推定分数}分（文字起こしのタイムスタンプから推定）

## 自動抽出項目
- 痛み（現状の不満）: {文字起こしから抽出}
- 理想（将来像）: {文字起こしから抽出}
- 障壁（決断を妨げる要因）: {文字起こしから抽出}
- 温度感: {高/中/低}
- 意思決定者: {本人/母/父/その他}

## SPIN評価スコアカード
| Phase | 内容 | 評価（5段階） | コメント |
|-------|------|:---:|----------|
| Phase 1 | アイスブレイク＋状況確認 | {1-5} | {評価理由} |
| Phase 2 | 痛みの深掘り | {1-5} | {評価理由} |
| Phase 3 | 理想の未来を描く | {1-5} | {評価理由} |
| Phase 4 | ギャップの確認 | {1-5} | {評価理由} |
| Phase 5 | サポート提案 | {1-5} | {評価理由} |
| Phase 6 | クロージング | {1-5} | {評価理由} |

## よかった点（3つ）
1. {具体的な発言や対応を引用}
2. {具体的な発言や対応を引用}
3. {具体的な発言や対応を引用}

## 問題点（3つ）
1. {具体的な問題と、なぜ問題かの説明}
2. {具体的な問題と、なぜ問題かの説明}
3. {具体的な問題と、なぜ問題かの説明}

## 改善アクション（3つ）
1. {次回から実行できる具体的なアクション}
2. {次回から実行できる具体的なアクション}
3. {次回から実行できる具体的なアクション}

## 過去データとの比較
- 成約者の平均面談時間（40分）との差: {差分}
- 最も近い過去パターン: {過去12件から最も類似するケース名}
- 類似点と相違点: {簡潔に}
```

## ステップ5: LINE文面4通生成

`面談後フォロー3-7-14ルール.md` のテンプレートを参照し、面談内容でカスタマイズしたLINE文面を生成する。

**保存先:** `ナレッジ格納庫/LINE下書き/{面談日}_{名前}.md`

**フォーマット:**

```
# LINE下書き: {名前}（{面談日}面談）

---

## 【当日】お礼LINE
送信タイミング: 面談終了後30分以内

{テンプレートA〜Dから温度感に合うものを選び、面談内容でカスタマイズ}

---

## 【3日後】フォローLINE
送信日: {面談日+3日}

{テンプレートE〜Gから状況に合うものを選び、カスタマイズ}

---

## 【7日後】期限通知LINE
送信日: {面談日+7日}

{テンプレートH〜Jから状況に合うものを選び、カスタマイズ}

---

## 【14日後】最終確認LINE
送信日: {面談日+14日}

{テンプレートK〜Lから状況に合うものを選び、カスタマイズ}
```

**選択基準:**
- 温度感「高」（仮申込済み）→ テンプレート A, E, H, K
- 温度感「中」（検討中）→ テンプレート B, F, I, L
- 温度感「低」（保護者相談必要）→ テンプレート C or D, G, J, L
- 各テンプレートの {面談要約}, {痛み}, {理想}, {障壁}, {名前} を面談内容で置換

## ステップ6: Asanaタスク作成

Asana MCPツールを使い、以下の構造でタスクを作成する。

**プロジェクト:** オールインワンプラン契約書（GID: `1209960384497212`）

### メインタスク
Asana MCP の `create_task_preview` でメインタスクを作成:
- タスク名: `{名前}さん 面談フォロー`
- 期日: {面談日+14日}（YYYY-MM-DD形式）
- 説明: **面談分析レポート全文を貼り付ける**（ステップ4で生成した内容）
  - 基本情報（名前・学年・同席者・面談時間）
  - 自動抽出項目（痛み・理想・障壁・温度感・意思決定者）
  - SPIN評価スコアカード
  - よかった点（3つ）
  - 問題点（3つ）
  - 改善アクション（3つ）
  - 過去データとの比較

### サブタスク4件
メインタスク作成後、そのタスクIDを使って4件のサブタスクを作成:

| サブタスク名 | 期日 | 説明 |
|-------------|------|------|
| 【当日】お礼LINE送信 | {面談日}（YYYY-MM-DD） | LINE下書きファイルの「当日」セクションを送信 |
| 【3日後】進捗確認LINE | {面談日+3日}（YYYY-MM-DD） | LINE下書きファイルの「3日後」セクションを送信 |
| 【7日後】期限通知LINE | {面談日+7日}（YYYY-MM-DD） | LINE下書きファイルの「7日後」セクションを送信 |
| 【14日後】最終確認LINE | {面談日+14日}（YYYY-MM-DD） | LINE下書きファイルの「14日後」セクションを送信 |
````
